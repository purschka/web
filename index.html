기본개념



WWW, W3 또는 웹(Web)이라고도 불리는 월드와이드웹(World Wide Web)은 다양한 형태의 데이터와 정보에 접근할 수 있도록 해 주는 인터넷 서비스다.

원래 월드와이드웹은 1989년부터 스위스 제네바에 위치한 유럽입자물리연구소(Organisation Européenne pour la Recherche Nucléaire, CERN)의 팀 버너스-리(Tim Berners-Lee)에 의해 개발되기 시작해 1991년 배포된 브라우저 이름이자, 고에너지 물리학계(High Energy Physics)의 국제적인 정보 및 자료 교환 프로젝트 모델이었다. 그 후 월드와이드웹은 보편적인 인터넷 서비스로 확대되었고, 1993년 모자이크(Mosaic)란 웹 브라우저(web browser)가 개발, 보급되면서 웹은 대표적인 인터넷 서비스로 폭발적인 발전을 거듭하여 현재에 이르고 있다.

여기서 인터넷의 개념을 먼저 살펴볼 필요가 있는데, 일반적으로 인터넷은 "TCP/IP라는 고유한 프로토콜을 기반으로 구축된 전 세계적인 네트워크들의 네트워크, 또는 이를 통해 구성되는 전 세계적인 사람들과 자원들의 집합체"라 정의된다.

인터넷 서비스의 하나인 월드와이드웹에 대해서는 다양한 정의가 있지만, 대체로 다음 네 가지 측면들이 포함되어야 할 것으로 보인다. 우선 첫 번째로 월드와이드웹은 인터넷 서비스다. 그것이 전 지구적 범위를 갖는다는 점도 인터넷 서비스로서 월드와이드웹의 속성을 규정하는 중요한 측면이다.

두 번째로 월드와이드웹은 기술적으로 HTTP(HyperText Transfer Protocol)라는 프로토콜을 사용하는 네트워크다.
세 번째로 월드와이드웹은 하이퍼텍스트 정보 시스템이다.
월드와이드웹은 기존의 인터넷 문서나 파일들과 달리 HTML(HyperText Markup Language)이라는 스크립트로 구성된 문서와 이들 문서들을 서로 연계시켜 주는 링크로 구성되는 웹 페이지들의 집합체라 할 수 있다.

네 번째로 월드와이드웹은 멀티미디어다. 월드와이드웹은 텍스트 정보뿐만 아니라 그래픽, 오디오, 비디오, 프로그램 파일 등을 하이퍼텍스트 형태로 제공한다.

최초의 웹 사이트

구조와 기술



웹 브라우저는 전 세계에 산재해 있는 웹 서버들로부터 HTML 문서들을 불러내 이용자가 읽을 수 있도록 클라이언트의 화면에 웹 페이지 형태로 디스플레이해 주는 하이퍼텍스트 리더(hypertext reader) 소프트웨어다. 웹이 출현한 이후 현재까지,최초의 브라우저인 버너스-리의 월드와이드웹에 이어 브라우저의 대명사라 불리는 모자이크, 넷스케이프, 인터넷 익스플로러, 그리고 최근의 구글 크롬(Google Chrome), 모질라 파이어폭스(Mozilla Firefox), 오페라(Opera) 등 수십 종의 브라우저들이 개발되어 왔다. 인터넷은 기본적으로 클라이언트-서버 모델(client-server model)을 기반으로 하는데, 서버는 네트워크를 통해 연결되어 있는 클라이언트의 요청에 따라 정보를 제공한다. 일반적으로 인터넷 서버는 그 서비스 기능에 따라 웹 서버(Web server), 메일 서버, FTP 서버, 유즈넷 뉴스 서버 등으로 나눌 수 있다. 웹 서버는 대개 HTML 문서로 되어 있는 웹 페이지들을 HTTP라는 프로토콜을 이용하여 웹 브라우저를 갖춘 클라이언트에 제공해 주는 기능을 담당하는 컴퓨터 하드웨어 및 소프트웨어 체계를 말한다. 특히 HTML 문서를 HTTP 프로토콜을 통해 제공하는 웹 서버를 HTTP 서버라고 부른다. 여기서 HTTP는 월드와이드웹에서 정보를 커뮤니케이션하는 가장 기본적인 프로토콜이다. 현재 HTTP의 표준은 팀 버너스-리가 설립한 월드와이드웹 컨소시엄(W3C, World Wide Web Consortium)에서 제정한다. 그리고 웹 사이트는 인터넷상의 월드와이드웹을 통해 접근할 수 있는 웹 페이지들의 모음이다. 우리에게 친숙한 책 은유(book metaphor)의 구성 요소를 그 이름으로 사용하는 웹 페이지는 HTML 포맷으로 되어 있는 문서 형태로서, 대개 하나의 파일 단위로 구성되어 있다. 일반적으로 웹 페이지 파일들은 웹 서버의 특정 폴더 및 하위 폴더에 위계적으로 저장된다. 웹 페이지는 텍스트 정보 이외에 그래픽, 이미지, 비디오, 사운드 등의 정보를 제공할 수 있다. 클라이언트 입장에서 웹 페이지는 웹 브라우저를 통해 사용자의 화면에 디스플레이된다. 이러한 웹 사이트의 웹 페이지들은 URL(uniform resource locator)이라는 인터넷상의 주소 체계를 통해 접근할 수 있다. 즉 모든 웹 문서들은 URL이라는 고유한 주소를 가지고 있다. 기존의 인터넷 서비스들과 달리 웹에서는 이와 같이 정보 접근의 보편성이 가능하다는 점에서 URL은 웹의 핵심적인 요소인 셈이다. 마지막으로 HTML은 월드와이드웹에서 정보를 표현하는 웹 페이지를 만들어 내는 데 사용되는 마크업(markup) 언어다. HTML은 월드와이드웹에 접속되는 컴퓨터에 텍스트 파일 형태로 저장되는데, 이 파일에는 단순 텍스트(plain text), 그리고 이 텍스트를 클라이언트의 웹 브라우저가 디스플레이하고 처리하는 데 필요한 명령들, 즉 마크업 등 두 가지 요소가 포함되어 있다.

역사



최초의 브라우저 또한 당연히 최초로 월드와이드웹을 개발한 CERN의 팀 버너스-리와 로베르 카이오에 의해 개발되었는데, 그 이름이 바로 공교롭게도 가상공간을 지칭하는 말과 동일한 월드와이드웹(WorldWideWeb)이었다. 당시 CERN에서는 월드와이드웹 이외에도 다양한 웹 브라우저들을 개발하였고, CERN 이외의 연구소들에서도 다수 브라우저들이 출시되었다. 그러나 초창기의 이와 같은 웹 브라우저들은 대체로 텍스트만을 표현할 수 있었고, 웹 사이트의 수가 적어 제한된 사용자들 사이에서만 이용될 수 있을 뿐이었다. 1993년 9월 일리노이대학교 NCSA(National Center for SuperComputing Applications)의 마크 앤드리센(Mark Andreesen)은 모자이크를 발표했는데, 이것은 설치하고 이용하는 데 편리할 뿐만 아니라 유닉스, 매킨토시, 마이크로소프트 윈도, IBM PC 등 모든 컴퓨터 환경에서 작동하는 범용 브라우저였다. 가장 중요한 점은 대개 텍스트만을 처리하던 기존의 브라우저와 달리 그래픽, 사운드, 비디오, 양식, 북마크, 파일 히스토리 등 다양한 기능을 지원한다는 점이다.
